(English Version) BEIDNA Whitepaper

Behavioral Economics Identity (BEI) – Triple-Helix Sovereign Civilization Infrastructure
Anchor: USPTO Application 19/383,599 (BEIDNA) | Since 1999 honeycomb namespace | FRAND-ready

Abstract

BEIDNA defines a triple-helix sovereign genome that encodes decentralized identity, verified behavior, and economic value into a single machine-checkable record. Each event emits a Tri-Header Receipt—Identity, Behavior, Economic—braided by construction rules and anchored on decentralized storage. A SovereignGeneCode binds lineage, consent, policy references, and merkle positions. An Admission Gate compiles ethics and regulation before issuance; FR2 bounded rollback executes deterministic correction with pre-funded reserves without deleting history. Public invariants—header integrity, lineage continuity, reserve sufficiency, jurisdictional compliance—allow third-party verification without private data. BEIDNA aligns with ISO 20022/4217, is FRAND-licensable, and applies to finance, healthcare, education, media, public records, and supply chains.

1. Vision

One identity, one mint, one clearing path per human; a structural encoding layer, not another app; century-scale continuity and ethics by design.

2. Historical Foundation (1999 → now)

An A–Z multi-system honeycomb namespace across countries, industries, interests, and time; hundreds of patents and thousands of claims; BEIDNA unifies them at the genome layer.

3. Core Invention
3.1 Triple-Helix Encoding

I-strand (DID, consent, revocation), B-strand (attestations, evidence, ZK proofs), E-strand (valuation, obligations, reserve locks). Receipts are braided at a common locus and hashed to a CID.

3.2 SovereignGeneCode

A parsable grammar that binds DID, policy lineage, and event topology, independent of database or UI.

3.3 Invariants & FR2 State Machine

Integrity, continuity, reserve sufficiency, compliance; roll within time/value caps using reserved collateral; append-only history.

3.4 Admission Score

Versioned, policy-compiled scoring with anti-gaming parameters at the encoding layer.

4. Distinction from prior BEI filings

Prior work focused on workflows for payments, clearing, credit. BEIDNA specifies the encoding algebra
encode / splice / fork / merge / roll / verify, enabling composability, formal verification, and SEP/FRAND pathways.

5. Honeycomb Namespace (Since 1999)

Cells = {Country/Region} × {Industry A–Z} × {Interest/Time}; each cell acts as a genome locus for attaching receipts and routing audits; supports Name/Store/Franchise NFTs.

6. Protocol & Data Model (Essentials)

7. Security, Privacy, Ethics

Selective disclosure, ZK-friendly proofs; Admission Gate; FR2 bounded rollback; auditable without private data leakage.

8. Industrial Applicability

Finance (TimeCoin, micro-remittance, chargebacks), Healthcare (CareMinutes, credentials), Education/Media (credentials, royalties, retractions), Public Benefits, Supply Chains (targeted recalls), Identity Commerce (store/franchise NFTs).

（中文版）BEIDNA 白皮书

行为经济身份（BEI）——三螺旋主权文明基础设施
锚点：USPTO 申请号 19/383,599（BEIDNA）｜自 1999 起的 A–Z 多系统蜂窝命名空间｜FRAND 标准就绪

摘要

BEIDNA 定义一种三螺旋主权基因组：把去中心化身份（I）、经验证行为（B）与经济价值（E）编码为单一、机器可验证的记录。每一事件产出三首部回执（Tri-Header Receipt）——身份首部、行为首部、经济首部——按编织规则合成并锚定于去中心化存储。SovereignGeneCode 绑定谱系、同意范围、政策引用与 Merkle 位置；Admission Gate 先验合规与伦理评分；FR2 有界回滚用预留准备金执行纠错且不删史。系统公开不变量（首部完整性、谱系连续性、准备金充足、法域合规），任何节点可在不取私数据的前提下验证。BEIDNA 可与 ISO 20022/4217 对齐，具备 FRAND 授权与标准采纳的可行性，适用于金融、医疗、教育、媒体、公共记录与供应链。

1. 愿景与前提

面向 83 亿人：一人一身份、一套铸造与清算路径。

从“流程”上升到“结构”：提供编码层而非又一个应用。

百年连续性：密码学与政策可版本化升级，无需重写历史。

伦理内置：发行前准入评分，发行后在 FR2 边界内纠错。

2. 历史起点（1999 → 今）

构建覆盖 A–Z 行业 / 各国地区 / 兴趣与时间 的蜂窝命名空间（多系统域名矩阵）。

数百份专利、数千项权利项沉淀为生态规范。

BEIDNA 把既有模块统一到可证明的基因层，形成文明底座。

3. 核心发明
3.1 三螺旋编码

I-Strand：DID、同意范围、撤销与恢复引用。

B-Strand：经验证行为、见证者集合、证据与 ZK 证明。

E-Strand：计价单位、义务与结算意图、准备金锁定。
三首部按编织规则在同一“位点”合成并哈希锚定（CID）。

3.2 SovereignGeneCode

可解析、可验证的基因级标识，把 DID、政策谱系与事件拓扑合一，独立于数据库与界面。

3.3 不变量与状态机

不变量：首部完整性、谱系连续性、准备金充足、法域合规。

FR2 状态机：时间/价值上限 + 预留准备金 → 触发 roll 纠错回执；历史只增不减。

3.4 准入评分（Admission Score）

评分函数族含区间、权重与反作弊参数；在编码层挂接版本化政策引用。

4. 与既有 BEI/清算专利的区别

既有文件聚焦支付/清算/信用等流程；BEIDNA 规定编码层代数
encode / splice / fork / merge / roll / verify，把流程变为可组合、可证明的结构，利于 SEP/FRAND 路径。

5. 蜂窝命名空间（自 1999）

Cell = {国家/地区} × {行业 A–Z} × {兴趣/时间}；每个 Cell 为基因位点：挂接回执、路由审计；支持 Name/Store/Franchise NFT 的命名与分润。

6. 协议与数据模型（要点）

IH：did, consent_scope, jur_code, key_ref, rev_ref

BH：attestor_set, action_type, evidence_ref, anti_abuse

EH：unit, price_ref, iso4217_code, reserve_lock, settlement

接口：上市（BEIGX）、清算（BEICX）、注册与标准（BEISX）、钱包与记账（ATMSBank）。

7. 安全、隐私与伦理
